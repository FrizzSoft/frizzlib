ItemPicker<T>

Abstract class which allows the listing of an array of items of type T. Classes that inherit from this class specify the type (T) of items to be listed, and the implementation details of the listing. Each item listed is given a number so that the user can select one of the items by its number. User can elect to display the items in batches of up to 100 items, and can type ? to get Help or ENTER to continue after each batch. User-defined messages can be displayed before and after each batch of items listed. Virtual methods are provided which can be over-ridden by the user to determine how each item is displayed in the list, and allow additional valid text responses.

PickItem(T[], int) pauses after each batch and accepts user input. The class abstracts away the recursion logic of the listing process.

The following methods may be overridden:

  • ItemFormatter(T) Logic to appropriately display one object to the console
  • IsAUserAllowedResponse(string?) Allows user to include additional allowed text responses
Pickers

Static class with methods that allow the user to select a folder using the Command Line Interface.

Available methods

  • PickFolder() - Prompt user to pick a folder, with initial folder listed being the UserProfile folder.
  • PickFolder(string) - Prompt user to pick a folder, specifying the initial folder to display.
  • PickFile() - Prompts user to pick a file, with initial folder listed being the UserProfile folder.
  • PickFile(string) - Prompts user to pick a file, specifying the initial folder to display.
